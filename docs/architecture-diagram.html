üé® Base + Overlays Pattern

This pattern allows you to define shared configuration once and override specific values per environment, following the DRY principle.

üìã
Base Values
Common configuration shared across ALL environments
image: repository: ecr.../myapp service: port: 80 targetPort: 8080
+
üîß
Overlay Values
Environment-specific overrides (dev/staging/prod)
image: tag: sha-abc1234 replicaCount: 3 ingress: host: myapp.prod.com
=
‚úÖ
Final Config
ArgoCD merges base + overlay for deployment
image: repository: ecr.../myapp tag: sha-abc1234 replicaCount: 3

üì¶ Generic Helm Chart Components

The generic-app chart is a production-ready template that includes all necessary Kubernetes resources with security and HA best practices built-in.

üöÄ Deployment

Manages pod lifecycle and rolling updates
  • Rolling update strategy
  • Configurable replicas
  • Resource requests/limits
  • Liveness & readiness probes
  • Security context (non-root)
  • Topology spread constraints

üîó Service

Internal load balancing across pods
  • Type: ClusterIP
  • Port mapping (80 ‚Üí 8080)
  • Selector-based routing
  • Service discovery via DNS
  • Session affinity (optional)

üìÆ Ingress

HTTP(S) routing from external traffic
  • nginx ingress class
  • Host-based routing
  • Path-based routing
  • TLS configuration (when enabled)
  • Annotations for customization

üë§ ServiceAccount

IRSA for AWS resource access
  • IAM role annotation
  • OIDC authentication
  • Least privilege access
  • No long-lived credentials

üìà HorizontalPodAutoscaler

Automatic scaling based on metrics
  • CPU-based scaling
  • Memory-based scaling
  • Min/max replica bounds
  • Target utilization
  • Optional (enable in prod)

ÔøΩÔøΩÔ∏è PodDisruptionBudget

  • minAvailable configuration

  • Prevents simultaneous evictions
  • Ensures HA during updates

üîÑ CI/CD Integration Workflow
1
Developer Commits Code

Push to develop branch in app-source repository triggers GitHub Actions CI pipeline

2
CI Builds & Pushes Image

GitHub Actions builds Docker image, runs security scan, pushes to ECR with tag sha-abc1234

3
CI Updates helm-charts Repo

Using GH_PAT, CI updates apps/myapp/overlays/dev/values.yaml with new image tag and commits

4
ArgoCD Detects Git Change

ArgoCD polls helm-charts repo every 3 minutes, detects drift (OutOfSync state)

5
ArgoCD Syncs to Cluster

Auto-sync policy triggers: ArgoCD runs helm template + kubectl apply to deploy new version

6
Kubernetes Rolling Update

New pods are created, health checks pass, old pods are terminated (zero downtime)
